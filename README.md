# Markdown 语法速查手册

Markdown 是一种轻量级标记语言，旨在让人们使用易读易写的纯文本格式编写文档，然后转换成有效的 XHTML（或 HTML）。

---

## 1. 标题 (Headings)
使用 `#` 号的数量来表示标题的级别（1-6 级）。

| 语法 | 效果 |
| :--- | :--- |
| `# 一级标题` |  |
| `## 二级标题` |  |
| `### 三级标题` |  |
| `#### 四级标题` |  |

> **注意**：`#` 与文字之间必须有一个**空格**。

---

## 2. 文本样式 (Text Styling)

| 描述 | 语法 | 效果 |
| :--- | :--- | :--- |
| **粗体** | `**粗体文本**` 或 `__粗体文本__` | **粗体文本** |
| *斜体* | `*斜体文本*` 或 `_斜体文本_` | *斜体文本* |
| ***粗斜体*** | `***粗斜体文本***` | ***粗斜体文本*** |
| ~~删除线~~ | `~~删除文本~~` | ~~删除文本~~ |
| `行内代码` | `` `代码内容` `` (使用反引号) | `代码内容` |
|  | `<u>下划线文本</u>` (HTML 标签) |  |
| ==高亮== | `==高亮文本==` (部分编辑器支持) | ==高亮文本== |

## 3. 列表 (Lists)

### 无序列表
使用 `-`, `+`, 或 `*` 加上空格。

```markdown
- 项目 1
- 项目 2
  - 子项目 2.1 (缩进 2 个空格或 1 个 Tab)
  - 子项目 2.2
```
**效果：**
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2

### 有序列表
使用数字加点 `.` 加上空格。

```markdown
1. 第一步
2. 第二步
3. 第三步
```
**效果：**
1. 第一步
2. 第二步
3. 第三步

### 任务列表 (Task Lists)
```markdown
- [x] 已完成的任务
- [ ] 待办任务
```
**效果：**
- [x] 已完成的任务
- [ ] 待办任务

## 4. 引用 (Blockquotes)
使用 `>` 符号。可以嵌套使用。

```markdown
> 这是一段引用文本。
> > 这是嵌套引用。
> 
> 引用可以包含**加粗**和*斜体*。
```
**效果：**
> 这是一段引用文本。
> > 这是嵌套引用。
> 
> 引用可以包含**加粗**和*斜体*。

---

## 5. 代码块 (Code Blocks)

### 行内代码
使用单个反引号 `` ` `` 包裹。
例如：使用 `npm install` 命令。

### 多行代码块
使用三个反引号 `` ``` `` 包裹，并可指定语言以实现语法高亮。

````markdown
```javascript
function hello() {
  console.log("Hello, World!");
}
```
````

**效果：**
```javascript
function hello() {
  console.log("Hello, World!");
}
```

支持的语言包括：`python`, `java`, `c`, `cpp`, `css`, `html`, `sql`, `json`, `bash`, `shell` 等。

## 6. 链接与图片 (Links & Images)

### 链接
语法：`[显示文本](链接地址 "可选标题")`

```markdown
[访问百度](https://www.baidu.com)
[带标题的链接](https://www.google.com "Google 搜索")
```
**效果：**
[访问百度](https://www.baidu.com)

### 图片
语法：`![替代文本](图片地址 "可选标题")`
*(注意前面有个感叹号 `!`)*

```markdown
![Logo](https://example.com/logo.png "网站 Logo")
```
**效果：**
(如果地址有效则显示图片，否则显示替代文本)

## 7. 表格 (Tables)
使用 `|` 分隔列，使用 `-` 分隔表头。`: ` 控制对齐方式。

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :----: | -----: |
| 单元格 | 单元格 | 单元格 |
| 文本   | 文本   | 文本   |
```

**效果：**

| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :----: | -----: |
| 单元格 | 单元格 | 单元格 |
| 文本   | 文本   | 文本   |

## 9. 转义字符 (Escape Characters)
如果你想显示 Markdown 的特殊符号本身（例如显示 `#` 而不是标题），请在符号前加反斜杠 `\`。

```markdown
\# 这不是标题
\* 这不是列表
```
**效果：**
\# 这不是标题
\* 这不是列表

## 10. 高级技巧 (Advanced)

### 脚注 (Footnotes)
```markdown
这是一个句子[^1]。

[^1]: 这是脚注的内容。
```
**效果：**
这是一个句子[^1]。
[^1]: 这是脚注的内容。

### 数学公式 (LaTeX)
大多数现代 Markdown 编辑器（如 Typora, Obsidian, GitHub）支持 LaTeX 公式。

*   行内公式：`$E=mc^2$` $\rightarrow$ $E=mc^2$
*   块级公式：
    ```markdown
    $$
    \sum_{i=1}^{n} x_i
    $$
    ```
    **效果：**
    $$
    \sum_{i=1}^{n} x_i
    $$

### HTML 标签
Markdown 兼容大部分原生 HTML 标签，当 Markdown 语法无法满足需求时（如改变字体颜色、大小），可以直接使用 HTML。

```html
<font color="red">红色文字</font>
<br> 换行
<details>
<summary>点击展开</summary>
隐藏的内容
</details>
```

## 💡 小贴士
1. **空行**：段落之间最好保留一个空行，否则可能会被渲染成同一段。
2. **兼容性**：不同的平台（GitHub, GitLab, 微信公众号, Notion）对 Markdown 的支持程度略有不同（特别是表格、脚注和高亮）。
3. **预览**：编写时随时使用预览功能查看最终效果。
